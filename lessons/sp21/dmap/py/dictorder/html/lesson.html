Dictionary Order

As you now know, the Python dictionary is just an unordered collection of key/value pairs. Since dictionaries are unordered collections, there's no need to build a dictionary in sorted order. In general there is no way to guarantee that the first item you add to a dictionary is the first item returned in the collection. This lesson is all about sorting the items in a dictionary.

Coder's Log A dictionary by definition is unordered. However in Python version 3.6 dictionaries will maintain insertion order. However, you should not rely on this behavior (your code could run by someone who is using 3.5).

Sorting

Since dictionaries have both keys and values AND since the values can be any valid Python data type, sorting is not as simple as sorting a list.

We will step you through a few of the common ways. It's not important to memorize the syntax; but you should know what you need to do to build a sorting process that reaches deep inside your dictionary.

Simple List Sorting

With a list you could just use the sorted function which works on lists (in ascending order -- the default):

values = [200, 100, 300]
print(sorted(values))
# normally you will have to type in the code given
values = [200, 100, 300]
print(sorted(values))

Reversing the Sort Order

If you wanted the order in descending order, you can just use the reverse parameter and set it to True.

values = [200, 100, 300]
print(sorted(values, reverse=True))

Sorting the dictionary keys

With dictionaries, you can sort the dictionary by keys:

simple = {
  "apples":  5,
  "pears":   2,
  "bananas": 12
}

for k in sorted(simple.keys()):
  print(k, simple[k])

As a reminder, the keys() method returns a view into the dictionary NOT a list of strings. Luckily, the sorted function takes an iterable as its parameter.

Sorting the dictionary values

You can also sort the dictionary values:

simple = {"apples":5, "pears":2, "bananas":12}

for v in sorted(simple.values()):
    print(v)

However, note that the method values() is only a set of values, at this point you no longer know which key is associated with which value.

Sorting the key, value pairs

To fix the problem noted above, you can ask the dictionary for all of its key/value pairs using the items() method. 

simple = {"apples":5, "pears":2, "bananas":12}
for t in sorted(simple.items()):
    # t is a tuple 
    # t[0] is the key, 
    # t[1] is the value
    print(t[0], t[1])

The sorted function is now working with a collection of tuples. The default value to use for sorting is the first item of the tuple (or sequence type). The key is at index 0 (the first item of the tuple), and the value is at index 1 (the second item of the tuple).

Using a helper function

Note that the dictionary is being sorted by the key (the name of the fruit).
If you want to sort by values instead, you have to create a special function to give to the sorted function (using the key named parameter):

Sorted by value, using the key

simple = {"apples":5, "pears":2, "bananas":12}
def by_value(key):
  # the key is a string (e.g. 'apples')
  # return the count 
  return simple[key]

for k in sorted(simple.keys(), key=by_value):
  print(k, simple[k])

The function by_value is called a helper function since it "helps" the sorted function find the values to use for comparison.

Sorted by value, using the item pair

Since items() returns a list of tuples, the sort function will be passed a tuple.
The sort function just returns the part of the tuple that holds the value.

simple = {"apples":5, "pears":2, "bananas":12}

def by_value(tuple_item):
  return tuple_item[1]

for t in sorted(simple.items(), key=by_value):
  print(t[0], t[1])

Since items is an iterator over key/value pairs, each item is then passed to the function by_value.
This function is receiving a tuple.

Complex Dictionaries

Just like any value inside a list can be any valid Python type (e.g. another list, a dictionary, a function, a number, a string, etc), the values of a dictionary can be any valid type as well.
In the example below the value for each key is another dictionary with two keys:

complex_map = {
   "apples":  {"count": 12, "calories": 200},
   "pears":   {"count": 2,  "calories": 100},
   "bananas": {"count": 5,  "calories": 300}
}
print(complex_map["apples"]["calories"])

Note that the keys of the complex dictionary are still strings or numbers, however, each value is yet another dictionary. So complex_map["apples"] returns a dictionary with two keys: count and calories. With this recursive definition (a dictionary inside another dictionary), you can build very complex and useful containers to hold our application data.

Complex Dictionaries, Complex Sorting

When the dictionary has a complex structure, sorting becomes complex. As mentioned before the sorting function can only work with very simple values. If you have a complex dictionary, you need to write a helper function that the sorting function will use to return the value to be used for sorting.

Let's look at an example. First you need to determine how you want to sort your dictionary. Let's sort the dictionary based on the number of fruits (the count key)
So you need to write a function that takes a dictionary key (a string) and returns the correct value to sort on. The function by_count shown below does that.

def by_count(key):
  return complex_map[key]["count"]

Now you can call the sorted function passing by_count as an argument to the sorted function: sorted(complex_map.keys(), key=by_count)

So here's what it looks like all together:

complex_map = {
  "apples":  {"count": 12, "calories": 200},
  "pears":   {"count": 2,  "calories": 100},
  "bananas": {"count": 5,  "calories": 300}
}

def by_count(key):  
  # note this function is accessing
  # a global variable (not good)! 
  return complex_map[key]["count"] 

for k in sorted(complex_map.keys(), key=by_count):
  print(k, complex_map[k])

Note the output. The outer dictionary is sorted by a value inside the inner dictionary.

Coder's Log The sorted function needs a list of things to sort. The helper function passed in is always given a single item from that list passed in. Your helper function is used to determine how to get a value that can be used for sorting from the item.

Fixing the 'global' sort

The problem with this sort is the sorting function by_count needs to access the variable that points to the dictionary (i.e. complex_map). In general, when you write functions, you don't want the function to access variables outside the scope of the function (i.e. global variables). It makes it difficult to test as well as to update and maintain the code.

As you saw, the dictionary method items() returns each of the dictionary components as (key, value) pairs as a tuple. The key is at index 0 and the value is at index 1.
Now the sorting function you pass into the sorted function (using the named parameter key) will accept a tuple (since that's what you are sorting):

def by_count(t):
  # t is a tuple 
  # t[0] is the key
  # t[1] is the value
  data = t[1]
  return data["count"]

for v in sorted(complex_map.items(), key=by_count):
  print(v)

Be sure you understand the above code.

For sorting dictionaries, you can work with the keys, values, or key/value pairs. In most cases you won't use the values() method, since there's no way to get the matching keys.

Revisiting Lambda Functions

Previously, you were introduced to the idea of using an inline lambda function to pass to the sorted function. As you may remember a lambda function is just an unnamed (sometimes called anonymous) function. As mentioned, a benefit of lambda functions is they don't pollute the name space of a Python program (the name space defines what variables and functions are visible to use). If a function is ONLY going to be used for a specific sorting purpose, there's no reason to make it visible to other functions.

You can use a lambda function to create an inline function.

for v in sorted(complex_map.items(), 
                key=lambda item: item[1]["count"]):
  print(v)

The great feature of lambda functions is that the dependence on the data is right there -- where it is being used. If you changed the structure of your 1dictionary, it's easy to figure out what needs to be updated in the respective sorting operation.

As a reminder, both functions below are functionally/semantically equivalent:

def is_odd1(x):
  return x%2 == 1

is_odd2 = lambda x : x%2 == 1
print(is_odd1(3) == is_odd2(3))

We will revisit lambda functions and even go into more details when we discuss functional programming.
