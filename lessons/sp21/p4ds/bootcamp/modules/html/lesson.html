Reusing Code

Python modules are a way to package up code (e.g. functions) in such a way that you can share/reuse the code across multiple projects. However, one of the main issues with notebook style platforms (colab, jupyter) is that they are not designed for this use case.

The main goal of this chapter is to highlight how to use modules (or libraries). We will also show the process of separating code into different files (or modules) so that you can easily reuse code. This is essentially how many of the popular Python and data science libraries (Numpy, Pandas, statistics, etc) are created.

We will also describe the process of how you can make your own modules once you do start writing code using a standard editor like Sublime Text or a full IDE (Integrated Development Environment) like PyCharm, Eclipse, Atom. However, in another lesson, we can show you how you can actually share the code of one notebook with another -- although this is not how it's usually done.

Reusing Code

Once you start building large programs, it helps to put functions into their own module/file in order to organize your code and to easily share it with others.
Once you create a module of your functions, you can import those functions into any program that could use them. This allows for code reuse.

Coder's Log Creating a re-usable Python module for large scale sharing involves more 'work' than we are showing here. However, the main concepts are exactly the same.

The first step of creating code that can be re-used in different projects is to put the code in a separate file -- sometimes called a module or library.

The following code is actually inside a file name LessonUtil.py. When you create a module, the file ends with .py. This file is available to your colab notebook when you install the IDE.

This module has the functions add and div. However, it also maintains a counter to keep track of how many times each of those functions are called. Take a look at its implementation:

# LessonUtil.py
#
# the modules lesson
#
def add(a, b):
    counters[add] += 1
    return a + b

def div(a, b):
    counters[div] += 1
    return a / b

def get_count(fn):
    return counters[fn]

# using functions as keys in a dictionary!
counters = {add: 0, div: 0}
def clear():
    for k in counters.keys():
        counters[k] = 0

importing code

Now once you have that code inside a separate file (LessonUtil.py) you can use Python's import statement to bring that code into your current project:

import LessonUtil

answer = LessonUtil.add(1,2)
print(answer)

In the above example, we are using the function add inside the module/library LessonUtil.py.

Shorthanding import as

You can also assign a name to your imported code to keep your typing short as well. Below we are using the same library but using the name util to access its contents:

import LessonUtil as util

answer = util.add(1,2)
print(answer)

Direct importing from

You can also import specific functions from a module as well. Here we use the syntax from moduleName import something.

from LessonUtil import add

answer = add(1,2)
print(answer)

This style of import comes at a cost as the first two ways shown provide a bit of documentation as to where add is coming from. In the previous example, the reader may not know where add is defined unless s/he read every import statement in the code.

Math

Python's math module contains many of the functions you might need to help with summing, trigonometry, logarithms, etc. At the end of this lesson is a list of available math functions that are useful. A simple example is shown in below:

import math

print(math.pi, math.e)

# sum up the numbers
numbers = [1,2,3,4,4]
print(math.fsum(numbers))

Please see https://docs.python.org/3.6/library/math.html for details (Yes, it is actually important that you read the documentation). Statistics

Python has its own statistics module, named stats; however, we will also be using the popular SciPy module for a fuller set of statistical functions. We will have separate lessons for these. Below shows some simple statistics functions Please read https://docs.python.org/3.6/library/statistics.html for details.

# statistics
import statistics
numbers = [1,2,3,4,4]
print("ave",  statistics.mean(numbers))
print("std",  statistics.stdev(numbers))
print("mode", statistics.mode(numbers))

Z Scores

A staple calculation in statistics is the z-score. As a quick reminder, a z-score is a measurement of how many standard deviations below or above a value is over the mean.
The z-score essentially rescales or normalizes your data. We saw an example earlier using the sciPy library to calculate it. For each value, you subtract the mean and divide by the standard deviation.

The default z-score calculated in the sciPy library considers the set of numbers to be the entire set from the population. As you may remember, when you sample data from a population, you need to adjust the standard deviation calculation (dividing by n - 1 instead of n) (we can simulate this later and show that calculating the sample variance using n - 1 is indeed an unbiased estimator for the population variance).
