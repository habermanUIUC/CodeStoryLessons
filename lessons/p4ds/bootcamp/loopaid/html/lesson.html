As we have learned, the for loop syntax is very straightforward:

for item in some_collection:
   do_something(item)

As we saw in the previous lesson, when you want to just add up a sequence, a while loop works well, but the syntax of using a for loop is a bit easier.
There's no need for managing a loop index and/or a boolean expression.

Looping over items in a collection (usually a list) is so common that Python also provides a few helper functions so that you can still use the for loop syntax in other situations where you might still want a loop index.

Home on the Range

The range function produces a sequence (not a list) of numbers that can be iterated over.
It is used to easily build a defined ordered set of numbers. You can even have a negative step value to count backwards. When you need a quick sequence of numbers, range is your answer.

range(start, stop, step)
  • start has a default value of 0
  • stop is required
  • step has a default value of 1

It will generate a sequence of values starting at start, ending at stop - 1, and each value in the sequence will be the previous value + step

The Range Function
for i in range(6):
    print(i)

When you use the range function (really a class) to create a 'virtual' list, the first parameter is the starting number and the second is the stopping point.

The stopping point is NOT included in the list (we say the end is exclusive). Also, the default step value is 1 (which can be changed by adding a third parameter):

backwards = range(4, 0, -1)
for n in backwards:
    print(n)

Stepping in Style

You can use the step parameter to generate specific sequences:

for i in range(2,12,2):
    print(i)

Negative Stepping

Going backwards is just as easy:

for i in range(12,0,-2):
    print(i)

Range as a sequence type

Using the range function returns a sequence (an immutable sequence) and you already know üòâ what you can do:

all_even = range(2,12,2)
print(len(all_even))     # length
print(all_even[0]))      # access
print(8 in all_even)     # membership
print(all_even.index(8)) # finding
print(all_even.count(8)) # counting

Range math

The math functions (min,max,sum) do work

print(min(all_even))   
print(max(all_even))   
print(sum(all_even))

Range to list conversion

In a few cases, you will need to convert a range type into a list. The range type is actually an immutable sequence -- you can't change any of the values in a range.

The following are some of the situations where you might want to convert it into a list (e.g. as_list = list(all_even)):

  • you want to print it
  • you want to use math operators with it (e.g. all_even * 3)
  • you want to compare it for equality

Range into list

Since the range function is a generator of items, you need to convert it into a list if you need full access to all the values for printing or to compare it to other lists.

r = range(0,3)
list(r) == [0,1,2]

Coder's Log Both built-in 'functions' list and range are actually types -- or classes in Python. Although they look like a function call, they actually create a Python object (or type). Python provides programmers the ability to build objects that return items as needed called a generator. Technically range is not a generator but it does generate numbers on demand, much like the generator concept which we will learn later.

Enumeration

Best of both worlds

If you find yourself needing to loop through items in a list/collection, but also need to know what index you are at, you can use the enumerate function which returns both the index and the item in the collection:

my_items = [1,2,3,'a']
for i, item in enumerate(my_items):
    if i >= 2:
        print(i, item)

There's a lot going on in that syntax. The function enumerate returns the index and the item from a collection of items (one set each time). Each time through the loop the following invariant is always true: item == my_items[i] i is also incremented (starting at 0).

Coder's Log an invariant is something that should always be true. It's something you can count on. The concept is used heavily in software design and testing.

Loop Variables

It is common to use the names i,j,k and idx for looping variables that are used as integers that represent the index into the list or array.

For both enumeration for for loops, the variable should be a singular noun version of the the variable being enumerated or looped over:

for prime in primes:
     print(prime)

Early Returns

Sometimes during looping, it possible that once you encounter a specific situation, you want to looping to stop. We will discuss in detail two Python keywords (break and continue) in another lesson, but as a reminder, if you add a return inside a loop, the function will immediately stop (and return a result).

Take a look at the code below and see if you can determine how it works:

def is_prime(n):
   i = 2
   while i < n:
      if n % i == 0
         return False
      i = i + 1
   return True

Iteration vs Sequences

As we have seen, loops and lists go hand in hand. Once again Python has generalized the concept of being able to process a set of items with the word iterable.
An iterable is anything (or type) that you can loop over (usually with a for loop). All sequence types are iterable and we will learn of some non-sequence types that are iterable as well.
