Strings and Lists

You've already been introduced to some of the most fundamental types in Python (numbers, booleans, strings, lists -- with only a few more to go). One of the hardest tasks of a new programmer is to remember what you can do with each type.

Two of the more common types are the string and the list; however, they have a common "parent" in their DNA: the sequence. The sequence 'class' belongs to many types and by learning what you can do with a sequence, helps you learn multiple types.

We begin this lesson with a few specific things you can do with strings and lists and then cover the common (the 'API') that apply to both.

There's a lot of information in this lesson. It is vital that you run and understand each example. In upcoming lessons and projects, knowing what you can do with both strings and lists will help your algorithm design. Many future lessons will use the information presented here.

Strings

We learned about the string type in an earlier lesson. Since many data analysis involves text, it's important to understand a few important operations that can be done on strings.

Creating Strings

The usual way to create strings is just assign a string value (called a literal) to a variable. However, you can also change values of other types into strings using the Python built-in function str.

# change a number into a string
x = str(8)
print(x, type(x))

Back to the String

As a side note, if a string represents a number, you can convert it to an actual number by using either the int or float functions:

print(int("121"), float("3.33"))

Strings from Lists

Almost all types provide some type of string representation. For lists of items, you can use the string's join method to glue the items in a list together:

parts = [1,2,3,4]
ssv = ';'.join(parts)
print(ssv)

Changing Strings

The following examples show how you can change or update the contents of a string. An important note (which is discussed in great detail in another lesson) is that all the methods shown actually return a new string and don't modify the target string.

Changing Case

Changing case is usually a data processing step in the data science pipeline where the analysis wants to treat both cases the same (called case insensitive). You can change the case of a string using the string methods lower and upper:

msg = "Hello Python"

# change to lowercase
print(msg.lower()) 

# change to uppercase    
print(msg.upper())

Coder's Log Functions vs Methods
This is the first time the word method has been used. A method is a function attached to a type (like lists, strings). When you create your own types (called classes), the functions that work on that type (or class) are called methods. To invoke a function method you use the . (dot) syntax: print("abc".upper()).

String Cleaning

There are several methods to 'clean' strings. Cleaning data will be discussed in great detail in upcoming lessons.

Removing Characters

The strip method also removes both leading and trailing whitespace (tabs, newlines, spaces):

msg = "  This is a story   "
print(len(msg), msg)

# remove whitespace at start and end
clean = msg.strip()
print(len(clean), clean)

The strip method also allows you to pass in a character to remove from the ends of a string.

msg = "I do love 'food'" 
print(msg.strip("'"))

Replacing Characters

You can also replace a character with another using the replace method:

# replace substrings
print(msg.replace('o', 'a'))

Finding Substrings

Strings have an additional method called find that returns the index of a substring. It returns -1 if the requested value is not found:

name = "Informatics"
print(name.find('for'))

You can even give a starting index to find to use:

name = "Python is Phun"
idx1 = name.find('P')
idx2 = name.find('P', idx2)

Splitting Strings

Many times data is encoded as a single string with separators. You can use the split method to pull out the different components and return a list of the different elements:

msg = "Apples are red. Bob likes apples."
parts = msg.split('.')
print(parts)

In the above example, a period was used to split msg into different parts. A common data format is separating data using a comma (csv - comma separated values):

csv = "12,11,10,0"
parts = csv.split(',')
print(parts)

You may notice that the empty string was returned in the first split example in the last component. This is because the string ended (terminated) with the character that was being used to split on. A few important notes:

  • the character used to split the string is not part of any of the returned strings.
  • if a string ends with the character that is being used to split the string, the returned array has an empty string as its last element.

Be sure to understand the above and this example:

csv = "12,11,10,0,"
parts = csv.split(',')
print(parts)

If you don't pass in a parameter to split, it will use the space, newline, and tabs (called whitespace) as the character (or token) to split on. You can easily get the words from a string this way:

msg = "Apples are red. Bob likes apples."
parts = msg.split()
print(parts)

Joining Strings

The opposite of spliting is joining. Once you have a list of items, you can also join string elements together to form a single string

ssv = ';'.join(parts)
print(ssv)

Lists

As discussed in the previous lessons, the list is one of the fundamental data structures in programming. Although we had much to say about the string, we will keep this section very brief on lists.

List Creation

In addition to creating a list using the bracket notation, you can also use the built-in function list to create a sequence of items -- much like str function, the list function. 

l = [1,2,3]     # matching square brackets
s = list("123") # function call to create a list

The Empty List

You can create empty lists using either as well:

e1 = []
e2 = list()

List methods

As we will see, the list is actually a type of sequence. In fact, the only list specific method is its ability to sort itself and we discuss sorting in a separate lesson.

print([3,2,1].sort())

Sequences

The string and the list belong to a broader (aka generic) class called the sequence. Every value that has type string or type list (e.g. print(type(list))) is also a sequence. We will also discuss other sequence types in upcoming chapters.

Sequence types provide a common set of operations that work for several different types of collections. It allows you, the software engineer, to design things (e.g. functions) to operate on a family of types rather than assuming a specific type.

Coder's Log Python provides a set of Abstract Base Classes (ABC) that formally define Protocols (a specific set of methods that must be implemented) that one can use to design new types. The Sequence is an ABC. We will dive deep into Python class design in the advanced class.

Immutable and Mutable Sequences

Sequence types are either mutable (e.g. lists) or immutable (e.g. strings). Mutable collections allow you to change the contents. Immutable sequences do not allow for changing any values in the sequence. We will discuss immutable sequences and why you will want to use them in a separate lesson.

As mentioned, the string is a sequence. It's just a sequence of characters. However, the string is an immutable sequence -- hence all the methods that 'changed' the underlying string actually return a new string.

All the fun stuff you can do with lists can also be done with strings. Part of this lesson is for you to explore the fact that the things you can do with any sequence type will work for both strings and lists.

Sequence Length

The len function is actually a built-in function in which you can pass in a sequence to determine its length (more on built in functions later). It returns the number of element in the sequence: 

l = [1,2,3]
s = "Python"
print(len(l), len(s))

Sequence Element Access

The nice feature of the sequence types is that you can access elements using the same syntax:

l = [1,2,3]
s = "Python"
print(l[1], s[1])

Sequence Membership

The in operator we used for list enumeration (for i in items) can also be used to ask a sequence if a value is contained within it:

l = [1,2,3]
s = "Python"

print(3 in l)
print('o' in s)

Sequence Element Finding

In addition to knowing that an item exists in a sequence, sometimes, you need to know where in the sequence it exists. The method index is one of the most useful methods of the sequence type. It returns (if the item exists) a value from 0 to len(list) - 1:

name = "Informatics"
print(name.index('c'))

l = ["abc", "def", "ghi"]
print(l.index("def"))

Note that the index throws an exception (an error) if you ask for something that does not exist (we will learn how to handle exceptions in another lesson):

def danger():
    l = ["abc", "def", "ghi"]
    print(l.index("hello"))
    
danger()

However, if you want to avoid dealing with the error, you can guard the index method with a membership test:

def safe():
    l = ["abc", "def", "ghi"]
    if 'abc' in l:
        print(l.index("abc"))
        
safe()

Sequence Element Counting

The count method on a sequence will provide a count of the number of those elements:

l = [1,2,3,4,3,2,1]
print("3's", l.count(3))
print("l's", "Hello".count('l'))

msg = "the quick fox ate the hot dog"
print(msg.count('t'))
print(msg.count('the'))

Sequence 'Math'

There are even some interesting things you can do with some arithmetic operators on sequences. Be sure to understand each example

Appending (concatenation) with +

The + operator can be used to append one sequence to another

# add to a string  
msg = "Hello Python"  
msg = " " + msg + ". " 
print(msg)

l1 = ['a','b','c']
l2 = ['z', 'y']
l3 = l1 + l2
print(l3)

l4 = [1,2,3]
l5 = [4,5,6,7]
l6 = l4 + l5   
print(l6)

a = [1,2,3]
b = ['a','b','c']
c = a + b
print(c)

Repetition/Replication with *

The * operator with sequences can be used to replicate. You can easily repeat a string value by multiplying it by a number (for easy string repetition):

print('H' * 5)

l1 = [1,2,3]
print(l1 * 3)

l2 = ['a','b','c']
print(l2 * 3)

a = [1,2,3]
b = ['a','b','c']
c = a * b
print(c)

Built-in functions

The following functions take a sequence sum, max, min (sum assumes a sequence of number types).

l1 = [1,2,3]
l2 = [4,5,6,7]

s1 = ['a','b','c']
print(min(l2), max(l2))
print(min(s1), max(s1))

print(sum(l1), sum(l2))

Sequence Equality

Much like comparing two values in Python to see if they are the same, you can use equality testing on sequences. For two sequences to be equal, they must contain the same values and be in the same order:

a = [1,2,3]
b = [3,2,1]
c = [1,2,3]
print(a == b, a == c)

Another part of sequence equality is that they have the same type. You might think that a list with the same characters as a string might be equal since they do share many of the same characteristics of equality:

word  = "hello"
chars = ['h', 'e', 'l', 'l', 'o']

print(word == chars)

When you create a list of characters using the list function, it is the same (although easier to type) as using the bracket notation:

s1 = list("abc")
s2 = ['a', 'b', 'c']
print(s1 == s2)

Forcing the Same Type

Sometimes it's easiest to use the list function to force any sequence into the list type (usually for equality testing):

Strings to lists

You can force the string into a list and then do equality:

word  = "hello"
chars = ['h', 'e', 'l', 'l', 'o']

print(word == ''.join(chars))
print(list(word) == chars)

Short Circuit Evaluation (reprise)

In the "selecting a path" lesson, there was a short discussion on short circuit boolean evaluation. This handy construct is useful for list manipulation as well. For example,

if items is None or len(items) == 0:
   # nothing to do

In the above example, len(items) is protected. It will not be evaluated unless items is not None. Since it's a logical or statement, as soon as true is evaluated, there's no need to process more. The function len will throw an exception if it is passed None. But using the above will protect that situation.

Similarly you can use the same to insist that a list has at least one item in it:

if items is not None and len(items) > 0:
   safe_access = items[0]

In the above example, the expression len(items) > 0 will only be evaluated if items is not None. Since it's a logical and as soon as a false is encountered, the processing can stop.
